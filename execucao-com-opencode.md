# Execução com OpenCode

## Decisão atual
O projeto mais ambicioso será desenvolvido com OpenCode como runtime experimental de implementação.

## Regras fixas
- OpenClaw continua como orquestrador principal
- o vault continua como fonte de verdade
- OpenCode entra apenas como runtime de execução
- a arquitetura do projeto não deve ser alterada por causa da ferramenta

## Como isso muda o plano
O foco deixa de ser apenas desenho estratégico e passa a incluir um caminho de bootstrap prático com OpenCode.

## Etapas recomendadas
### 1. Preparação
- validar instalação do OpenCode
- criar setup do projeto
- garantir AGENTES.md e contexto do projeto

### 2. Bootstrap
- ler README do projeto
- ler visão do produto
- ler MVP
- ler arquitetura
- ler spec técnica

### 3. Primeiro corte do produto
- escolher uma fatia inicial pequena e postável
- implementar com layout base e núcleo funcional mínimo
- validar antes de abrir próxima frente

## Regra operacional
Mesmo usando OpenCode, continuar trabalhando em fatias pequenas, com spec-first e validação explícita.
