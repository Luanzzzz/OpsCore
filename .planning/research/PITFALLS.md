# Pitfalls Research: OpsCore

**Context:** Risk synthesis from current vision and scope documents.
**Date:** 2026-04-17

## Pitfalls

| Pitfall | Warning Signs | Prevention | Phase |
|---------|---------------|------------|-------|
| Trying to build every channel too early | Scope explodes around integrations before core workflow exists | Build around a normalized internal intake model first | Phase 5 |
| Making IA features vague and hard to verify | Requirements say "use AI" without concrete operator benefit | Tie each IA feature to category, urgency, summary or next action outcomes | Phase 1 / Phase 4 |
| Splitting inbox, tasks and agenda into disconnected tools | Users must re-enter data between modules | Preserve links between entities from the first conversion flow | Phase 2 / Phase 3 |
| Overweighting dashboard polish before operational truth exists | Pretty metrics without trusted workflow data | Build dashboard from real queue/task states, not placeholder KPIs | Phase 1 / Phase 4 |
| Letting OpsCore collapse back into InboxFlow | Scope narrows to triage-only experience | Maintain task, agenda and orchestration requirements in roadmap | All phases |

## Highest-Risk Area

The biggest product risk is losing cohesion: either by becoming too broad to execute or too narrow to preserve the platform thesis.
