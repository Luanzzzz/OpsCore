<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cliente/API futura -> queries agenda | payloads de agenda podem tentar forjar origem, status ou data |
| queries agenda -> store JSON | dados persistidos viram source of truth local |
| agenda -> inbox/tasks | snapshot cruza fronteira entre dominios e deve vir de leitura server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | T | `createAgendaItem` | mitigate | validar `linkedType`/`linkedId` contra `getInboxItemById` ou `getTaskById` e montar snapshot no servidor |
| T-03-02 | T | `src/lib/validation/agenda.ts` | mitigate | validar enums, id positivo, textos curtos e `dueAt` ISO antes de persistir |
| T-03-03 | T | `createAgendaItem` | mitigate | impedir duplicata ativa exata por origem, tipo e vencimento |
| T-03-04 | R | `updateAgendaStatus` | mitigate | registrar timeline minima de criacao, reagendamento, conclusao e cancelamento |
| T-03-05 | D | `.data/agenda-items.json` | mitigate | persistir snapshot minimo e notas limitadas; nao aceitar blobs livres que ampliem crescimento file-backed |
</threat_model>
