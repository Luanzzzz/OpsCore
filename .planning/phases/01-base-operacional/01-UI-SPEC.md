# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | lucide-react or equivalent linear icon set |
| Font | IBM Plex Sans |

**Direction:** manual design system, web-first, focused on dense operational UI rather than marketing surfaces or card-heavy layouts.

---

## Primary Screen Contract

| Element | Contract |
|---------|----------|
| Primary screen | Inbox operacional |
| Focal point | Lista densa de entradas com prioridade, status e ultima atividade visiveis sem abrir o detalhe |
| Secondary anchor | Painel de detalhe do item selecionado |
| Tertiary zone | Dashboard-resumo com poucos blocos operacionais |
| Navigation model | Sidebar compacta + header funcional + area principal em two-pane layout no desktop |
| Mobile fallback | Lista em tela principal; detalhe abre como tela dedicada ou drawer de tela cheia |

**Visual hierarchy:**
1. Linha selecionada e seus sinais operacionais (`Critica`, `Aguardando resposta`, envelhecimento do item)
2. Painel de detalhe com resumo contextual e sugestoes de IA
3. Radar operacional com gargalos e itens prioritarios

**Accessibility note:** qualquer acao com icone isolado deve ter label textual no hover/focus e `aria-label` explicita.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Separacao entre badge, ponto de status e micro-indicadores |
| sm | 8px | Gaps internos de linha, labels e acoes compactas |
| md | 16px | Espacamento padrao entre blocos e padding de componentes |
| lg | 24px | Padding de paineis, header interno, secoes de detalhe |
| xl | 32px | Gaps entre colunas e grupos principais |
| 2xl | 48px | Separacao entre zonas principais da pagina |
| 3xl | 64px | Respiro vertical de paginas e estados vazios |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.3 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.1 |

**Type rules:**
- Maximo de 4 tamanhos e 2 pesos.
- Numeros-chave do dashboard podem usar `Display` apenas em blocos prioritarios.
- Linhas do inbox usam `Body` para titulo e `Label` para metadados compactos.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F4F1EA | Background geral, canvas principal, areas de respiro |
| Secondary (30%) | #D9E2D3 | Sidebar, paines secundarios, cards/resumos operacionais |
| Accent (10%) | #C65D2E | CTA primaria, item ativo da navegacao, destaque de foco, contadores realmente prioritarios |
| Destructive | #A63A2F | Acoes destrutivas, confirmacoes de arquivamento/exclusao, erros criticos |

Accent reserved for: `Registrar entrada`, estado ativo da navegacao, foco de teclado, badge `Critica`, contadores de urgencia, highlight do item selecionado.

**Semantic support colors:**
- Success/state stable: `#4F6D5A`
- Warning/waiting: `#B78A2A`
- Neutral text strong: `#243126`
- Neutral text soft: `#5A6B5F`

**Color behavior:**
- Nao usar accent em todos os botoes secundários.
- `Aguardando resposta` usa warning, nao accent.
- `Concluida/Arquivada` usa neutro dessaturado.

## Component and Layout Contract

| Area | Contract |
|------|----------|
| Inbox list | Linhas densas, altura compacta, sem cards empilhados como estrutura dominante |
| Detail panel | Secoes claras: contexto, sugestoes de IA, historico essencial, proximos passos |
| Dashboard blocks | 4-5 blocos maximos, orientados a acao, sem excesso de graficos |
| Filters/sort | Barra superior funcional com ordenacao e filtros visiveis |
| Badges | Usados para prioridade, status e origem; sempre com contraste alto |
| Empty states | Devem empurrar o usuario para a proxima acao, nunca apenas informar ausencia |

---

## Interaction and States

| State | Contract |
|-------|----------|
| Default list | Operador consegue varrer rapidamente prioridade, status e ultima atividade |
| Selected item | Painel de detalhe atualiza sem perder contexto da fila |
| Loading | Skeletons discretos em lista, detalhe e blocos do dashboard |
| Empty inbox | Mostrar CTA primario para registrar a primeira entrada |
| Error | Explicar o problema e oferecer caminho claro de recuperacao |
| IA suggestion pending | Placeholder curto indicando que sugestoes estao sendo preparadas |
| IA suggestion ready | Categoria, urgencia, proxima acao, resumo e justificativa curta aparecem juntos |

**Interaction priorities:**
- Clique principal abre ou seleciona item.
- Duplo clique nao e necessario para o fluxo.
- Ordenacao e filtros devem ser utilizaveis sem menu escondido complexo.

---

## Motion Contract

| Motion | Rule |
|--------|------|
| List/detail transition | 120-180ms, easing suave, sem overshoot |
| Badge/status updates | Transicao discreta de cor/opacidade, sem animacoes decorativas |
| Dashboard refresh | Atualizacao suave com fade leve ou count-up curto |

**Do not use:** animacoes grandes, parallax, shimmer excessivo, microinteracoes chamativas sem funcao operacional.

---

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Registrar entrada |
| Empty state heading | Nenhuma entrada operacional ainda |
| Empty state body | Registre a primeira entrada para iniciar a fila, receber triagem com IA e montar o radar operacional. |
| Error state | Nao foi possivel carregar a fila operacional. Tente atualizar a tela ou revise os dados da entrada mais recente. |
| Destructive confirmation | Arquivar entrada: confirme somente se este item realmente saiu do fluxo ativo da operacao. |

**Tone:** claro, operacional, sem linguagem de help desk genérico e sem jargao corporativo vazio.

**Secondary labels recommended:**
- `Ordenar por prioridade`
- `Filtrar por status`
- `Aguardando resposta`
- `Sugestao de proxima acao`
- `Justificativa da IA`
